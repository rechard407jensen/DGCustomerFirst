## Contribution Guidelines

- Fork the repository and clone it locally.
- Create a new branch for each significant change.
- Follow existing code patterns and naming conventions.
- Write clear, descriptive commit messages.
- Test your changes thoroughly before submitting.

## How to Contribute

1. Fork the repository.
2. Clone your fork using `git clone`.
3. Create a new branch using `git checkout -b feature-name`.
4. Make your changes locally.
5. Commit your work using `git commit -m "your message"`.
6. Push to your fork using `git push origin feature-name`.
7. Open a Pull Request against the main repository.

## Reporting Issues

- Open a new issue describing the bug.
- Include reproduction steps, screenshots, or error logs.
- Check existing issues before posting a new one.

## Suggesting Features

- Open an issue with the label `enhancement`.
- Clearly describe the feature and its use-case.
- Suggest alternatives if applicable.

## Code of Conduct

Please be respectful and inclusive. We follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/version/2/0/code_of_conduct/).

## Review Process

Pull Requests will be reviewed within 7 days. Feedback may be provided. Once approved, your changes will be merged into the main branch.
